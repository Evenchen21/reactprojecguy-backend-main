# 🚀 פרויקט צד-שרת לכרטיסי ביקור 🪪

ברוכים הבאים לתיעוד של צד השרת של אפליקציית כרטיסי הביקור. מערכת זו מספקת API יציב ומאובטח לניהול כל הלוגיקה העסקית של הפרויקט, החל מאימות משתמשים ועד לניהול כרטיסי הביקור עצמם.

## ✨ מה המערכת יודעת לעשות? (פונקציונליות)

המערכת מחולקת לשני מודולים עיקריים:

- **👤 ניהול משתמשים:**
  - **הרשמה ואימות:** מאפשר למשתמשים חדשים להירשם למערכת. הסיסמאות מוצפנות באופן אוטומטי כדי להבטיח אבטחה מקסימלית.
  - **התחברות מאובטחת:** משתמשים רשומים יכולים להתחבר ולקבל טוקן (JWT) המאפשר להם לגשת לאזורים המוגנים של המערכת.
  - **מערכת הרשאות מבוססת תפקידים:**
    - **משתמש רגיל:** יכול לצפות בכרטיסי ביקור ולנהל את רשימת המועדפים שלו.
    - **משתמש עסקי:** בנוסף להרשאות של משתמש רגיל, יכול ליצור, לעדכן ולמחוק את כרטיסי הביקור של עצמו.
    - **מנהל (Admin):** בעל גישה מלאה לכל הפעולות במערכת, כולל ניהול כל המשתמשים וכל הכרטיסים.

- **📇 ניהול כרטיסי ביקור (CRUD):**
  - **יצירה (Create):** משתמשים עסקיים ומנהלים יכולים ליצור כרטיסי ביקור חדשים.
  - **קריאה (Read):** כל המשתמשים (גם אורחים לא רשומים) יכולים לצפות בכרטיסי הביקור.
  - **עדכון (Update):** משתמשים עסקיים יכולים לעדכן רק את הכרטיסים שהם יצרו. מנהלים יכולים לעדכן כל כרטיס.
  - **מחיקה (Delete):** משתמשים עסקיים יכולים למחוק את הכרטיסים שלהם. מנהלים יכולים למחוק כל כרטיס.

## 🛠️ באילו כלים השתמשנו?ה (טכנולוגיות)

הפרויקט בנוי על בסיס טכנולוגיות מודרניות המאפשרות פיתוח מהיר, יציב ומאובטח:

- **סביבת הרצה:** **Node.js** - מאפשרת להריץ קוד JavaScript בצד השרת.
- **פריימוורק לשרת:** **Express.js** - שלד פופולרי לבניית שרתי Web ו-API, המקל על ניהול נתיבים ובקשות.
- **מסד נתונים:** **MongoDB** - מסד נתונים NoSQL גמיש המאפשר שמירת נתונים בפורמט דמוי JSON (נקרא BSON).
- **ניהול מסד נתונים:** **Mongoose** - ספרייה המפשטת את העבודה עם MongoDB ומאפשרת להגדיר סכמות ומודלים למידע.
- **אבטחה ואימות:**
  - **JWT (JSON Web Tokens):** שיטה להעברת מידע מאומת ומאובטח בין צד הלקוח לשרת.
  - **bcryptjs:** ספרייה להצפנת סיסמאות בצורה חזקה לפני שמירתן במסד הנתונים.
- **ולידציה:** **Joi** - כלי לווידוא שהמידע המתקבל מהלקוח (למשל, בטופס הרשמה) תואם לכללים שהוגדרו.
- **כלים נוספים:**
  - **CORS:** מנגנון המאפשר לשרת לקבל בקשות מדומיינים אחרים (למשל, מאפליקציית ה-React שרצה בפורט אחר).
  - **dotenv:** כלי לניהול משתני סביבה (כמו מפתחות סודיים וכתובות) בקובץ נפרד ולא ישירות בקוד.

## 🎮 איך מריצים את הפרויקט? (התקנה והרצה)

כדי להפעיל את השרת על המחשב שלכם, בצעו את השלבים הבאים:

1.  **דרישות קדם:** ודאו שיש לכם **Node.js** ושרת **MongoDB** מותקנים ורצים על המחשב.
2.  **שכפול הפרויקט:** הורידו או שכפלו את קוד המקור למחשב שלכם.
3.  **התקנת תלויות:** פתחו טרמינל בתיקיית הפרויקט והריצו את הפקודה שתתקין את כל הספריות הנדרשות:
    ```bash
    npm install
    ```
4.  **הגדרת משתני סביבה:**

- העתיקו את הקובץ `.env.example` לקובץ `.env`.
- עדכנו את הערכים בקובץ `.env` לפי סביבת העבודה שלכם:

  ```
  # הגדרות פורט השרת
  PORT=8000

  # כתובת החיבור למסד הנתונים של MongoDB
  DB_HOST=mongodb://localhost:27017/bcard_project

  # מפתח סודי להצפנת טוקנים (JWT) - מומלץ להחליף במחרוזת ארוכה ומורכבת
  JWT_SECRET_KEY= xxxxxxxxxxxxxxxxx
  ```

5.  **הרצת השרת:**
    - להרצה רגילה, השתמשו בפקודה:
      ```bash
      npm start
      ```
    - **מומלץ לפיתוח:** הריצו את השרת במצב פיתוח עם `nodemon`:
      ```bash
      npm run dev
      ```

6.  **טעינת נתוני התחלה (Seed):**
    - לאחר שהמסד רץ וקובץ `.env` מוגדר, אפשר להכניס משתמשים וכרטיסים לדוגמה:
      ```bash
      npm run seed
      ```
    - לחלופין, אפשר להריץ סקריפט לדוגמת כרטיס בודד:
      ```bash
      npm run seed:card
      ```

## 🔌 נתיבי משתמשים חשובים

- הרשמה: `POST /api/users`
- התחברות: `POST /api/users/login`

וזהו! השרת שלכם באוויר, מאזין על הפורט שהוגדר, ומוכן לקבל בקשות. 🎉
